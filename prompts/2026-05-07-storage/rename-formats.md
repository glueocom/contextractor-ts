# Rename `txt` → `text`; add `original` (raw page HTML) format

Rename the `txt` format identifier to `text` across the full stack (Rust → TypeScript → CLI → Actor). Add `original` as a new saveable format that writes the raw Playwright-captured HTML, bypassing Trafilatura. Accept `txt` as a backward-compat alias that silently normalises to `text`.

## File-by-file changes

### `packages/extraction/native/src/lib.rs`
- `OutputFormat::Txt => "txt"` → `"text"` — the string the Rust side returns to TypeScript callers
- Input side stays: `"txt" | "text" | "plain" => Ok(OutputFormat::Txt)` — no change needed
- Update JSDoc comment in `extractMetadata` and `extractAllFormats`: `"txt"` → `"text"`, `["txt", ...]` → `["text", ...]`
- Run `cargo test --workspace` and `cargo clippy --workspace --all-targets -- -D warnings` after

Auto-generated by napi-rs — fix the JSDoc comments in `lib.rs` (which feed the generator) rather than editing this file by hand.

## `original` format in the Apify Actor

The actor already saves raw HTML via `saveRawHtmlToKeyValueStore` → `saveHtml` flag in `createApifySink` → `result.html`. No format-level changes needed for `original` in the actor. The existing mechanism is correct.

## Acceptance criteria

- `--save text` writes `{slug}.txt` (extension stays `.txt`)
- `--save txt` also works (alias normalises silently)
- `--save original` writes `{slug}-raw.html` with raw Playwright-captured HTML
- `--save all` includes `text` and `original`
- Apify Actor: `saveExtractedTextToKeyValueStore: true` continues to save plain text
- `pnpm test` passes; `cargo test --workspace` passes; `cargo clippy --workspace --all-targets -- -D warnings` passes
