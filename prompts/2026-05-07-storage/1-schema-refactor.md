# Schema Refactor: Unified `save` and `saveDestination` Fields

## Context

Replace the four boolean save fields in the Actor input with a single `save` enum array (matching the CLI `--save` style) and a new Actor-only `saveDestination` field. As part of this, rename the `txt` format identifier to `text` across the full stack (Rust ŌåÆ TypeScript ŌåÆ CLI ŌåÆ Actor) and add `original` as a new saveable format for raw page HTML. Source of truth: `packages/schema/src/input.ts`.

The npm package (`@contextractor/standalone`) is both a CLI tool and a Node.js library ŌĆö it exports a programmatic API in addition to the binary.

No backward compatibility is required. Break any existing API, CLI flags, Actor input, or Docker interface without hesitation ŌĆö this is a clean cut.

Fields to remove:

- `saveRawHtmlToKeyValueStore`
- `saveExtractedTextToKeyValueStore`
- `saveExtractedJsonToKeyValueStore`
- `saveExtractedMarkdownToKeyValueStore`

## Format Rename: `txt` ŌåÆ `text`; Add `original` Format

Rename the `txt` format identifier to `text` across the full stack. Add `original` as a saveable format that writes raw Playwright-captured HTML (bypasses Trafilatura). No backward compatibility ŌĆö `txt` is removed entirely.

**Scope**: Rust output string (`lib.rs`), TypeScript `OutputFormat` type, `SaveFormat` in CLI, all sinks, CLI help text, tests, Actor schema, Apify sinks.

## Parameter and Naming Review

Before finalizing any schema field, CLI flag, or dataset property, audit every name against May 2026 industry standards and API design best practices:

- Are names consistent across the input schema, CLI flags, and dataset output?
- Do names follow the dominant convention for the context (camelCase for JSON/TS, kebab-case for CLI flags)?
- Are names unambiguous, self-documenting, and free of abbreviations that have clearer alternatives?
- Are enum values lowercase and hyphen-separated where appropriate?
- Are boolean field names framed as positive assertions (avoid `disableX` ŌĆö prefer `skipX` or restructure)?
- Are grouped concepts named with a consistent prefix or suffix?

Rename any field that fails this review. No backward compatibility is required.
