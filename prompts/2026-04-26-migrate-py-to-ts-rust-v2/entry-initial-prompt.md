## Lessons from the v1 implementation pass (avoid repeating)

- `apify push` deploys to whatever actor is named in `.actor/actor.json`'s `name` field under the logged-in org. To target `glueo/contextractor-test`, the file MUST contain `"name": "contextractor-test"`. v1 left it at `"name": "contextractor"` and accidentally pushed build 0.3.5 to **production `glueo/contextractor`**. Set the test name from step one and add a check (CI grep, or a pre-push assertion) so a prod push is impossible without an explicit override.
- `rs-trafilatura` is **0.2.x** (not 0.1.x as old notes assumed). The `Options` struct has no `prune_xpath`, no `tei_validation`, no `with_metadata` flag (metadata is always extracted), and `output_markdown` defaults to `false` (must be set per call to get `content_markdown`). `content_html` is always populated. `Metadata` exposes `categories`, `tags`, `license`, `image`, `page_type` that the Python `MetadataResult` did not. Mirror the actual struct, not the Python config.
- rs-trafilatura's metadata title heuristic differs from Python `trafilatura` (it sometimes returns the meta description). Tests must assert metadata is non-empty / matches a regex, not exact-string equality, or they will be brittle for no good reason.
- napi-rs `#[napi]` macros read return-type tokens literally â€” `use napi::bindgen_prelude::Result as MyResult` produces broken `.d.ts` (the type alias name leaks). Use the bare `Result<T>` from `napi::bindgen_prelude::Result` everywhere a `#[napi]` fn returns one.
- `tsconfig.json` `exactOptionalPropertyTypes: true` is incompatible with napi-rs-generated optional fields (it emits `field?: T`, not `field?: T \| undefined`). Leave it off in the root tsconfig, or build option-objects conditionally.
- A virtual Cargo workspace with empty `members = []` fails `cargo metadata`. The first scaffolding step must already create the napi-rs crate dir with at least a stub `Cargo.toml` so the workspace parses.
- `vitest run` exits 1 when a package has zero `*.test.ts` files. Apps without tests (e.g. `apps/contextractor-apify`) need `vitest run --passWithNoTests` in their `test` script, or the recursive `pnpm -r test` fails.
- Biome's default file scope is too broad. Explicitly ignore `.claude/**`, `prompts/**`, `**/fixtures/**`, `**/test-suites/**`, `**/test-suites-output/**`, `**/*.node`, and the napi-rs-emitted `packages/contextractor-engine/native/index.{js,d.ts}` from `biome.json`.
- Local prerequisites: Rust toolchain via `rustup` (cargo + rustc on PATH for napi build), Apify CLI **â‰Ą 1.4** (older versions reject the modern actor format with "Actor is of an unknown format"), Node 22+, pnpm 10+. Document these in CLAUDE.md and the engine README.
